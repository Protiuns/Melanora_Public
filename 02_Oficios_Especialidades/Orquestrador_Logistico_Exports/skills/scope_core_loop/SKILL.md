# Game Scope - Core Loop & Fun

O escopo de um jogo deve ser construído de dentro para fora. Antes de planejar 50 níveis, você deve garantir que o "Core Loop" (o ciclo básico de 30 segundos) é divertido.

## 1. Isolando o Ciclo Principal
O Core Loop é a ação repetitiva fundamental.
- **Exemplo RPG**: Andar -> Encontrar Inimigo -> Lutar -> Ganhar Loot -> Subir de Nível.
- **Exemplo Puzzle**: Ver o tabuleiro -> Analisar padrão -> Mover peça -> Limpar linha.

**Tarefa**: O seu escopo deve priorizar a perfeição desse ciclo. Variações (novos inimigos, cores diferentes) são escopo secundário.

## 2. A Regra do MVP (Mínimo Viável)
O seu protótipo deve ser divertido usando apenas cubos e esferas (Greyboxing).
- Se clicar e pular não for prazeroso sem arte, o jogo tem um problema de design, não de escopo.
- **Diversão vem da mecânica**, não da casca. Gaste 70% do seu tempo de escopo inicial polindo a mecânica básica.

## 3. Os 4 Pilares do Jogo
Verifique se o seu escopo contempla:
1. **Participação Voluntária**: O jogador escolhe estar ali.
2. **Regras**: O que é proibido e o que é permitido.
3. **Objetivos**: Fim claro para o esforço do jogador.
4. **Sistema de Feedback**: Como o jogo diz ao jogador: "Você está indo bem/mal" (sons, brilhos, barras de vida).

## Conclusão
Se o Core Loop não está validado, qualquer escopo adicional (história, cosméticos) é desperdício de recurso.
