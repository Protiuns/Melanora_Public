# Game Scope - Content Verification (SGDD)

O Método SGDD (Short Game Design Document) foca na objetividade. Esta etapa garante que você tenha uma visão completa de todos os "objetos físicos" que compõem o seu jogo antes de começar a produzir.

## 1. Fluxo Completo (A Experiência Linear)
Descreva o jogo como se estivesse contando uma história para alguém.
- **O Início**: Como o jogo abre? Qual a primeira interação?
- **O Meio**: Qual o ciclo de gameplay? Como o jogador progride?
- **O Fim**: Como o jogo termina? Qual a tela final?

> **Dica**: Se você não consegue descrever o fluxo em um parágrafo, o jogo ainda é uma "nuvem" de ideias, não um projeto.

## 2. A "Lista de Compras" de Assets
Transforme o seu texto de fluxo em uma lista técnica dividida por categorias:
- **Arte**: Sprites, Tilesets, Modelos 3D, Animações, Backgrounds.
- **Áudio**: Músicas (SFX), Ambientes (BGM), Efeitos de Interface.
- **Programação**: Sistemas de movimento, IA de inimigos, Sistemas de Save, Máquinas de Estado.

## 3. Interfaces "Invisíveis"
Não esqueça das telas técnicas que o jogador espera encontrar:
- **Menus**: Splash Screen, Menu Inicial, Pausa, Créditos.
- **Feedback**: Tela de Vitória/Derrota, Tutorial, Indicadores de UI (HP, Stamina).
- **Configurações**: Volume, Resolução, Idioma.

## Conclusão
O escopo está "completo" nesta etapa quando a soma desses assets e sistemas cabe na sua capacidade de produção atual.
