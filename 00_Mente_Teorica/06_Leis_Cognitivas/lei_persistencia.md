# 💾 Lei da Persistência de Relevância (L0)

Esta lei governa o que a Melanora deve "lembrar" e "esquecer", protegendo o Banco de Dados Sináptico contra a poluição de dados insignificantes.

## 1. Critério de Eternização
Um insight ou dado só deve ser persistido no Banco de Dados se:
- **Impacto Sistêmico**: Alterar a forma como dois ou mais especialistas colaboram.
- **Frequência de Falha**: Representar um padrão de erro recorrente (mais de 2 vezes).
- **Inovação Axiomática**: Representar uma nova lei ou princípio aprovado pelo Newton.

## 2. Poda de Memória (Pruning)
Memórias que não são acessadas pelo Analista Matemático por mais de 3 ciclos de auditoria devem ser marcadas como **Irrelevantes**.

## 3. Critérios de Irrelevância para Marcação
O sistema deve marcar para revisão humana qualquer elemento que:
- **Obsolescência Temporal**: Arquivos de log ou insights de tarefas que foram concluídas há mais de 30 dias sem nova referência.
- **Isolamento Sináptico**: Arquivos `.md` com zero links de entrada e saída por mais de 2 auditorias consecutivas.
- **Redundância Cognitiva**: Informações que foram sintetizadas em um arquivo de "Theoria" ou "Axioma" superior (o original torna-se rastro).

## 4. Estado de "Quarentena Analítica"
Elementos marcados não são apagados. Eles entram em uma lista no `synaptic_database.json` aguardando a decisão do Newton:
- **Deletar**: Remoção física permanente.
- **Arquivar**: Movimentação para `99_Arquivo_Morto/`.
- **Reintegrar**: Atualização dos links para reativar o elemento.
